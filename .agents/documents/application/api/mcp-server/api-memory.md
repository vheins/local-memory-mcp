# API Specification: Memory (MCP Tools & Resources)

## 1. Endpoints (Tools)

### 1.1 `memory.store`
- **Method:** `tools/call`
- **Arguments:**
  - `type` (string, enum: code_fact, decision, mistake, pattern, agent_handoff, required)
  - `title` (string, required)
  - `content` (string, required)
  - `importance` (number 1-5, required)
  - `tags` (array of strings, optional)
  - `is_global` (boolean, optional)
  - `scope` (object { repo: string }, required)
- **Response:**
  ```json
  {
    "content": [{ "type": "text", "text": "Memory stored..." }],
    "structuredData": { "id": "uuid-v4-string" }
  }
  ```

### 1.2 `memory.search`
- **Method:** `tools/call`
- **Arguments:**
  - `query` (string, required)
  - `repo` (string, optional)
  - `limit` (number, optional)
  - `minImportance` (number, optional)
- **Response:**
  ```json
  {
    "structuredData": {
      "results": [{ "id": "...", "title": "...", "content": "...", "relevance": 0.95 }]
    }
  }
  ```

### 1.3 `memory.synthesize`
- **Method:** `tools/call`
- **Arguments:**
  - `query` (string, required)
  - `repo` (string, required)
- **Requirement:** Client must support `sampling` capability.

### 1.4 `memory.acknowledge`
- **Method:** `tools/call`
- **Arguments:**
  - `memory_id` (string/UUID, required)
  - `status` (string, enum: used, irrelevant, contradictory, required)

### 1.5 `memory.recap`
- **Method:** `tools/call`
- **Arguments:**
  - `repo` (string, required)
  - `limit` (number, optional)

## 2. Resources

### `memory://index`
- Returns a JSON array of all memory IDs and titles for the current context.

### `memory://summary/{repo}`
- Returns the high-level summary of the repository generated by `memory.summarize`.
